# Приложение "Список задач"

🚀 **Простое и удобное приложение для управления задачами** с поддержкой темной темы, локализацией и сохранением данных в браузере.

[![Демо](https://img.shields.io/badge/ДЕМО-Онлайн-brightgreen)](https://gamesport-todo-list.netlify.app/)
[![Репозиторий](https://img.shields.io/badge/GITHUB-Репозиторий-blue)](https://github.com/danielgreen229/gamesport-todo-list)

## ✨ Возможности

✅ **Добавление, редактирование и удаление задач**  
✅ **Фильтрация задач** (Все/Активные/Завершенные)  
✅ **Поиск по задачам**  
✅ **Приоритеты задач** (Высокий/Средний/Низкий)  
✅ **Темная/Светлая тема** (сохранение в `localStorage`)  
✅ **Локализация** (Английский/Русский)  
✅ **Импорт/Экспорт задач** (формат JSON)  
✅ **Пагинация**  
✅ **Плавные анимации**  
✅ **Адаптивный дизайн**  

## 🛠 Технологии

- **Vue 3** (Composition API)
- **Pinia** (Управление состоянием)
- **Vue Router** (Навигация)
- **Tailwind CSS** (Стилизация)
- **Vitest** (Тестирование)
- **Vite** (Сборка)

## 🚀 Быстрый старт

### 🖥️ Mock-сервер API

Для разработки и тестирования используется mock-сервер на базе `json-server`, который имитирует REST API:

- **Расположение**: `server/db.json`
- **Порт**: 3001
- **Запускается** автоматически вместе с dev-сервером (`npm run dev`)
- **Функционал**:
  - CRUD операции с задачами
  - Поддержка стандартных REST эндпоинтов
  - Сохранение состояния между перезапусками
  - Реализация задержек ответа (для тестирования loading states)

Пример структуры данных в `db.json`:
```json
{
  "todos": [
    {
      "id": 1,
      "text": "Пример задачи",
      "done": false,
      "priority": "medium",
      "createdAt": "2024-01-01T00:00:00.000Z"
    }
  ]
}
